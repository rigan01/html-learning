Document and website structure

The main definitions that we have to understand:

<main>
To use once per page, do not nested within other elements, is for content unique to this page.
<article>
For example a single blog post.
<section>
It is similar to article element, but it is more for grouping together a single part of the page that constitue a single piece of functionality(i.e. a mini map, or a set of article headlines and summaries).It is recommending to begin each section with a heading.
<aside>
Content that is not directly related to the main content(author biography, related links, etc.).
<header>
It can be the child of the body(global header of the page), or the child of the section and article(specific header for that section).
<nav>
Represent the main navigation functionality for the page.
<footer>
Represent of end content for the page.

Non-semantic wrappers

When we need to just grouping a set of elements together, then we can use the <div> and <span> elements that is preferable to use these with a suitable class attribute.

SPAN element

Span is a inline non-semantic element, use the span element only if can't think of a better semmantic text element to wrap your content.For example, in this case span element is used to just provide a class to put the red color for the text wrapped in.

DIV element

Div element is a non-semantic block level element, which you should use only if you can't think of a better semmantic block element to use.Take care with divs, it's easy to use them too much, and after that you will have hard time to updating and maintaining your documents.
